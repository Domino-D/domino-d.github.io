@media

语法：

@media mediatype and|not|only (media feature) {
    CSS-Code;
}

mediatype 常用参数all ； screen
and|not|only not不常用；only可以放于mediatype前做限制；and不能前置与media feature相连，同时也用于多个media feature之间相连。
media feature min-width：定义输出设备中的页面最小可见区域宽度；max-width：定义输出设备中的页面最大可见区域宽度；orientation：portrait|landscape分别表示竖屏、横屏控制。
CSS-Code 编写对应状态下的css

用法：

打印样式
@media print {
color: red;
}
手机等小屏幕手持设备
@media screen and (min-width: 320px) and (max-width: 480px){
body {
background: yellow;
}
}
平板之类的宽度 1024 以下设备
@media only screen and (min-width: 321px) and (max-width: 1024px) {
body {
background: blue;
}
}
PC客户端或大屏幕设备: 1024px 至更大
@media only screen and (min-width: 1024px) {
body {
background: green;
}
}
竖屏
@media screen and (orientation:portrait) and (max-width: 720px) {对应样式}
横屏
@media screen and (orientation:landscape){对应样式}

@import

是css引入的另一种方式，且只能引用css文件。他是在html页面和link标签中css文件加载完成后，再远程加载调用@import xx.css，会引起浏览器回流、重绘和加载，产生http连接，页面加载时长增加，非常不推荐使用。还是用link标签来的实在。

@keyframes

CSS 中 animation属性函数定义

语法：

@keyframes animationname {
keyframes-selector {
css-styles;
}
}

animationname 函数名可自定义
keyframes-selector 方式一：0%-100%{css-styles} ；方式二：from{css-styles} to{css-styles}

用法：

@keyframes

根据不同浏览器内核，添加相应前缀，如@-webkit-keyframes

0%-100%方式
@keyframes mymove {
 0%   {top:0px;}
 25%  {top:200px;}
 50%  {top:100px;}
 75%  {top:200px;}
 100% {top:0px;}
}
@-webkit-keyframes mymove {
 0%   {top:0px;}
 25%  {top:200px;}
 50%  {top:100px;}
 75%  {top:200px;}
 100% {top:0px;}
}
from{css-styles} to{css-styles}方式
@keyframes mymove {
 from {top:0px;}
 to {top:0px;}
}
@-webkit-keyframes mymove {
 from {top:0px;}
 to {top:0px;}
}

:before和:afterCSS中的伪元素，

可以实现在现有的DOM元素前后增加内容，行内联元素，本身不作为DOM元素，不能用JS进行DOM操作，点击伪元素相当于点击现有元素本身。

添加内容作用（不赘述）

清除浮动作用

.clearfix:after {
content: '';
clear: both;
}

其中的 ‘.clearfix’ 是类名称，这里只是举例说明

content属性值允许为空值 但是 content属性必须写上 ，否则伪元素不生效，清除浮动也不会生效。
相对于空div标签增加清除浮动的方法，伪元素清除浮动的方法，不产生dom节点，这也就不利于搜索和错误检查。在删除清除浮动功能时，空标签容易被发现，需要格外注意检查是不是有伪元素清除浮动作用，避免影响其他元素布局。

height: 100px;
