node.js 安装

软件下载地址：http://nodejs.cn/download/
根据IOS / Windows系统下载对应版本。 后续将针对Windows系统下的情况进行说明。
软件安装时比较简单，下一步到底，建议直接在默认路径下进行安装，方便后续安装全局模块。
node.js集成了npm，安装好node.js后npm也就一并安装好了。

node.js和npm安装版本确认

win + R呼出 快速启动命令窗口 ，输入cmd，回车，进入 命令提示符窗口 。
输入node -v，回车，显示node当前安装的版本号，如：v8.11.1
再输入npm -v，回车，显示npm当前安装的版本号，如：5.6.0
如若两个版本号都能正常显示，表明node.js安装完毕。

在项目路径下安装模块组件

1、cmd命令指示符指向项目路径下：

先将路径定义到计算机分盘区域下，如D盘，实现方法：在cmd中输入d:，回车
随后将路径定义到项目文件夹下，如D:\reacttest，实现方法：在cmd中输入cd reacttest,回车。

2、项目文件初始化即package.json文件生成

2-1、在cmd中输入npm init，回车，完成相关描述的配置，包含如下几项：
name：默认显示为项目文件夹名称，如果需要修改名称，在行内直接键入名称，之后回车
version：根据情况更改，命名规则为“主版本号.次版本号.修订号”，如：1.0.0
description：描述
main：默认为index.js项目主文件
keywords：描述项目的关键字
scripts：申明npm脚本指令
repository：项目代码存放地址，如对应github repository url
author：作者
license：遵循协议
上述内容编写完成后cmd显示确认信息（yes/no），直接输入yes确认
2-2、或者还有一种简略（ 懒 ）方法生成package.json文件，在cmd中输入npm init -y，回车。后面如果有需要，可以手动配置填写。

3、导入依赖

主要有两种依赖，运行环境的依赖（dependencies） 和 开发环境的依赖（devDependencies）。
3-1、如果有现成的依赖代码，直接写入package.json中并保存，随后在cmd中将路径定义在项目文件夹下，输入npm install，系统会自动搜索项目文件夹根目录下的package.json，并依据dependencies和devDependencies下列出的依赖包一一加载。多人协同的项目下，能够一次性加载所需依赖，不用一个一个的输入，比较便捷。如果需要将项目上传托管，整module文件夹是不需要上传的，只需要将packa.json文件上传上去就好了，实际项目开发要用的时候，按照如上方法一次性导入依赖包
3-2、Package.json文件初始化生成后，代码中没有dependencies和devDependencies两项内容。项目文件夹中也只有package.json一个文件。相应的依赖包需要一一加载导入。
3-3、依赖导入的具体方法
dependencies依赖导入方法：
1、打开cmd将路径定义在项目文件的根目录下
2、输入npm install –save xxx，回车，xxx是依赖包名称，如：react，react-dom等
devDependencies依赖导入方法：
1、打开cmd将路径定义在项目文件的根目录下
2、输入npm install –save-dev xxx，回车，xxx是依赖包名称，如：webpack等

依赖导入支持连写，以空格隔开依赖包名称，如npm install –save react react-dom一次性导入多个依赖包。

导入依赖包时遇到的 坑 ：

a.关于babel-preset-es2015

导入babel-preset-es2015依赖包时，目前该依赖包还是会正确安装的到对应的module文件夹下，但是安装完成后，cmd会提醒建议安装对应的升级替代依赖包babel-preset-env。可以直接使用babel-preset-env代替babel-preset-es2015

b.关于webpack

webpack拆分出一个单独的webpack-cli的依赖包，目前单独导入webpack是不够的，在项目编写完成后执行webpack命令进行打包的时候，cmd会显示提醒需要导入另外一个webpack-cli依赖包，否则webpack命令将不会执行完成。

c.关于webpack.config.js

简要配置方法下（不包含loader，plugins）：


//webpack.config.js（使用的是node语法，不要点击IDE提示的ES6转换，否则打包时会出错）
var path = require(‘path’);
var react = require(‘react’);
module.exports = {
entry: dirname + “/app/main.js”,//唯一入口文件,从webpack 4.x版本后，默认入口文件为 ./src/index.js
output: {   //唯一出口文件,从webpack 4.x版本后，默认出口文件为 ./dist/main.js
path: dirname + “/public”,//打包后输出文件所存放的文件夹名称
filename: “build.js”//打包后输出文件的文件名称
},
devServer: {
},
module: {
},
plugins: {
}
};
//__dirname是nodejs中的全局变量，指向当前脚本根目录，及项目文件根目录

关于babel依赖包

babel是起代码转换作用，可以转换ES6语法，允许在开发过程中使用ES6编写而不必担心后续的代码转换
常用的babel依赖包有: babel-preset-env babel-preset-react babel-core
babel-loader babel-plugin-transform-runtime
Babel依赖包在npm命令安装注册后要进行相应的简单配置。在项目根目录中新建文件，并命名为’.babelrc’ 千万不要忘记‘.’ , 配置代码如下：


// .babelrc
{
presets: [’env’,’stage-0’,’react’],  //babel-preset-xxx 对应项下的都罗列一遍即可
plugins: [’transform-runtime’]  //babel-plugin-xxx 对应项下的都罗列一遍即可
}

关于webpack-dev-server

webpack-dev-server依赖包，实时编译打包，在计算机内存中生成出口文件main.js，保存在项目根目录下，但是并不可见，即在物理内存中是不显示的。可以有效避免开发编译过程中，为了实时浏览结果而不停的读取物理硬盘内存，造成卡顿乃至硬盘损伤。
依赖包安装在开发环境中，即通过npm i webpack-dev-server –save-dev命令安装依赖
依赖包安装完成后，在 webpack.json 文件中进行配置，在script标签下添加"dev": "webpack-dev-server --open",不要忘记上一行的,


“scripts”: {
    “test”: “echo \”Error: no test specified\” && exit 1”,
    “dev”: “webpack-dev-server –open” //表示运行 npm run dev 后，直接执行打包，并且在完成打包后，开启默认浏览器预览结果
}

关于html-webpack-plugin

允许计算机自动编译出口文件，不占物理内存。
首先导入依赖包cnpm i html-webpack-plugin -D
随后要在webpack.config.js中进行配置：


//webpack.config.js
const path = require(‘path’)//引入path
const HtmlWebPackPlugin = require(‘html-webpack-plugin’) //引入html-webpack-plugin插件
const htmlPlugin = new HtmlWebPackPlugin({
    template: path.join(__dirname,’./src/index.html’), //定义模拟出口路径，不占物理内存
    filename: ‘index.html’ //定义模拟出口文件名称，不占物理内存
})

静态父子组件传值

静态组件传值方法不适用于路由挂载组件，通过路由挂载形式的父子关系组件进行传值，可以使用 react-redux 来统一管理组件数据传递问题。

refs属性的使用

React提供了一个很好用的 ref 属性方法。最常用在 <input/> 标签中的数据双向绑定的功能实现上。（说起来还是要诟病一下，react的双向数据绑定功能实现起来，真的真的没有vue来的直接，个人很喜欢vue中的 v-model 双向数据绑定方法，使用起来也确实是简单粗暴省时省力。）重点开始，<input/> 标签中的值可以用 e.target.value 的方法来获取，react提供的 ref 方法是直接给 <input/> 标签赋予一个 ref 属性，其值为字符串，例：ref='_string'通过 this.props.refs._string.value 来获取 <input/> 标签中的值。

value与onChange

在 <input/> 标签中实现数据双向绑定的方式，实际运用起来相对vue还是有点复杂。 <input/> 标签如果只是绑定数据那么可以使用 defaultValue 属性来实现，还有一种是 placeholder 属性方法，个人理解为是一种预填值的方法，做为默认值提示来使用，如果在该栏目下输入某一个值，便会覆盖掉 placeholder 的默认值。当需要实现 <input/> 标签数值双向绑定的功能时，不能使用 defaultValue 属性，需要直接使用value属性，而且必须同时绑定onChange方法，如果只有value属性而没有绑定onChange方法，系统会报错。

因此，简单来说， value 属性和 onChange 方法是相辅相成的，两者共同配合才能实现真正的数值双向绑定功能。这里记录一点，在input标签中同时使用 value 属性和 placeholder 属性是不冲突的，两者的实际效果也不会对冲，合理合规。

异步函数中的调用实例组件this（即this指向暂存）

关于避免this指向性错误的正确打开方式。

在使用异步函数之前，如果涉及到异步函数访问当前组件的 this 属性的情况，在异步函数之前将当前组件的 this 暂存到异步函数外部包裹函数的作用域中，通常可定义为：const that = this，如此一来，在异步函数中直接调用that来间接指向所需的当前组件的this属性。例如：axios异步调用，访问数据，并运用返回结果更改react组件构造器中state里面的数据，在axios中直接调用that进而间接指向当前组件this属性，进而完成业务功能。

axios 中patch和delete方法

在axios中提供了默认的 get、post 方法，并且还额外提供了 patch、delete 等方法。在实际项目中发现，简单的axios.patch()和axios.delete()方法并不能直接的访问后端对应的patch和delete方法。从根本上来讲，patch和delete方法执行机制还是默认的两种方法，即get和post的变种。因此，可以直接用原始的 get、post 方法实现功能，不纠结。

绑定点击事件书写规范

添加点击事件的正确书写方法：onClick={() => this.myfunction()}

需要在实际调用函数外面包裹一个匿名钩子函数。如果需要访问点击事件，写法如下：

onClick={e => this.myfunction(e)}

如此，e被传入函数内部，可以用于执行阻止默认事件或者阻止事件捕获和冒泡的方法。

如果是在遍历函数map中调用item到某个组件函数内部，写法如下：

onClick={() => this.myfunction(item)}

前面的中括号中就不要写item了，写了会报错，而且不能正确传值。
