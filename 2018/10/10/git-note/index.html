git-note

发表于 2018-10-10

git add <filename.extension>

工作区到暂存区的提交

git commit -m ““

暂存区到版本仓库的提交，由用户自定义描述

git diff – <filename.extension>

比较分支下某个文件的差异，文件名称需要标明扩展名

git status

展示工作区和暂存区当前状态

git log –pretty=oneline

单行显示日志，不包含版本回退和删除日志

git reflog

显示日志，包含所有日志。退出q

git reset –hard __

git reset –hard HEAD^

git reset –hard HEAD~[1-100]

git reset –hard

版本回退或跳转

git reset HEAD <filename.extension>

回退到提交至暂存区前的状态，文件名称需要标明扩展名

cat <filename.extension>

打开当前文件，展示和编辑源代码

git checkout – <filename.extension>

放弃暂存区文件状态变动，文件名称需要标明扩展名

git checkout -b [branch name]

新建分支

git checkout [branch name]

切换到指定分支

git rm <filename.extension>

删除工作区问价，文件名称需要标明扩展名

git remote add origin [url]

添加远程仓库连接

git push origin [local branch name] : [remote branch name]

推送本地代码到远程仓库， –force 强制推送

git pull origin [remote branch name] : [local branch name]

拉取远程仓库代码到本地

git merge [branch name]

将指定分支合并到当前所在分支，如有冲突手动解决

git stash

现场暂存

git stash list

暂存列表

git stash apply stash@{‘num’}

回复到暂存列表中的某一暂存现场

git stash pop

回到暂存现场，并删除暂存列表内容
